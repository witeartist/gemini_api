# Security — Wite AI

## Аутентификация

### Пароли
- **Алгоритм:** PBKDF2 (100 000 итераций, SHA-256, 32-byte salt)
- **Автомиграция:** старые SHA-256 хеши автоматически обновляются при логине
- **Хранение:** `data/users.json` (хеш + salt, пароль никогда не хранится в открытом виде)

### Сессии
- **Bearer-токены** в заголовке `Authorization`
- **TTL:** 7 дней
- **Персистентность:** сессии сохраняются в `data/sessions.json`, переживают перезапуск
- **Автоочистка:** истёкшие сессии удаляются при старте и периодически

---

## Шифрование

### API ключи
- **Алгоритм:** AES-256-CBC
- **Ключ шифрования:** генерируется автоматически и хранится в `data/.encryption_key`
- Альтернативно: переменная окружения `KEY_ENCRYPTION_SECRET`
- Ключи шифруются при сохранении, расшифровываются при использовании

### Пользовательские API ключи
- Хранятся в `localStorage` браузера
- **Не отправляются на сервер** — используются для прямых запросов к API
- Каждое устройство требует отдельной настройки

## Защита от атак

### Path Traversal
- Все `userId` и пути проходят валидацию: только `[a-zA-Z0-9_-]`
- Проверка `../` и символов выхода из директории
- Все файловые операции ограничены `DATA_DIR`

### Rate Limiting
| Тип | Лимит |
|-----|-------|
| Общий API | 200 запросов / мин |
| Логин | 15 попыток / 15 мин |

### CORS
- **Production:** отключён (same-origin через Nginx)
- **Внешняя галерея:** CORS включён для `/api/external_gallery` и `/api/files/`
- **Dev mode:** разрешён `localhost:3000`

### Прочее
- **trust proxy** — корректное определение IP за Nginx
- **Валидация входных данных** — проверка типов, размеров, форматов
- **Graceful shutdown** — корректное завершение при SIGTERM/SIGINT

## Рекомендации

1. **Не коммитьте `data/`** — добавлен в `.gitignore`
2. **Бекапьте `data/.encryption_key`** — без него серверные API ключи нельзя расшифровать
3. **Используйте HTTPS** в production (Let's Encrypt + Nginx)
4. **Обновляйте зависимости** — `npm audit` периодически
5. **Ограничьте доступ** — Nginx `allow`/`deny` для `/api/admin/`
